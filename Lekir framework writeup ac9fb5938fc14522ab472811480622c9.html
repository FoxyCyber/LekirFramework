TOOLS :

  1. SQLMAP

Next, we have to craft the payload. Why? We want to find the vulnerable part of this web. Head over to your terminal and go into your lekir file. Launch the sqlmap command and you’ll see it appears like so :

We have to input the (sqlmap -u “url” -p <parameter to test> --dbs) command in the terminal.

if our case, it’ll be

sqlmap -u "http://localhost:1337/sqlinjectionlow.php" -p user_id --dbs

You can see here, that is doesn't work. Why?

Let’s circle back to the source code to figure out why it doesn't work.

After checking the source code, it seems that the user_id parameter was requested using a post request. Which means, we need to specify that in the body of the request using the --data option.

Now, we go back to the terminal and execute the new line command :

$ sqlmap -u http://localhost:1337/sqlinjectionlow.php -p user_id --data "user_id=1" --dbs

Unfortunately, like you see the arrow, we can’t use user_id parameter since it is not injectable. Then I remembered that when you want to access Lekir Framework you need to be logged in right? Let’s check our cookies.

Let’s head back to the browser and inspect our developer tools at the cookies tab back.

Take note of the PHPSESSID since we cant use user_id as our parameter. Then, head over to the storage tab and you can see the PHPSESSID is there. Take the value of the PHPSESSID and put it in the new command line for the sqlmap tool. The new input would be :

sqlmap -u http://localhost:1337/sqlinjectionlow.php --data "user_id=1" --cookie "PHPSESSID=595662d4011cc9d79635359134cde693c" --dbs

Since we can run this can we can see it is indeed vulnerable, we can move to the other procedure, “Enumeration” . This means the process of systematically probing a target for information.

  1. ENUMERATION

Our mission is to see if there’s any valuable data in this framework. So we’ll access each of the database. To do that, we just have to add the previous lines with “--tables” like so :

sqlmap -u http://localhost:1337/sqlinjectionlow.php --data "user_id=1" --cookie "PHPSESSID=595662d4011cc9d79635359134cde693c" --dbs --tables

Then, a bunch of table lists is displayed. Analyze each tables and we stumble across :

Secret? What could it be? Could it be a valuable data in this database? Let’s access it. How? We will use --dump. just incorporate it in the previous command.

sqlmap -u http://localhost:1337/sqlinjectionlow.php --data "user_id=1" --cookie "PHPSESSID=595662d4011cc9d79635359134cde693c" --dbs -T secret --dump

we have access the file, but there’s only a string in here.. It looks like a base64 encoded data.. We have to decode this.

Go back to lekir framework, we have the base64 decoder in the tools tab. Go there and input the string we found in the terminal there.
